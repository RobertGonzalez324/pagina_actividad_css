Informe de la actividad

El sitio es estático y no usa backend. Todas las páginas son HTML y se enlazan desde el menú.

La página de acceso está en contacto.html y guarda la sesión en el navegador usando localStorage.

Archivos principales:
- index.html: inicio con presentación del tema.
- nosotros.html: información del proyecto.
- servicios.html: secciones de contenido y tarjetas.
- contacto.html: login y registro local.
- css/estilos.css: estilos generales del sitio.
- js/main.js: lógica de login y registro.

Cómo funciona:
- HTML crea la estructura de cada página.
- CSS define colores, diseño en cuadrícula, botones y tarjetas.
- JavaScript guarda y lee datos de localStorage para el acceso.

Problemas y soluciones:
- No hay backend. Se usa localStorage.
- Diseñar varias páginas iguales. Se usa un solo CSS compartido.
- Validar el acceso. Se usan required en HTML y comprobaciones en JavaScript.

El proyecto está listo para entregarse con los archivos necesarios y sin backend.
